# 锦鲤前程 — 后端计划书

## 一、架构总览

```
┌──────────────────────┐      ┌──────────────────────┐
│  微信小程序前端       │      │    后端服务 (back/)    │
│  (pages/utils/*.js)  │◄────►│  Node.js + Express     │
│                      │REST  │  + Socket.IO           │
│  现有：               │      │                        │
│  - localStorage 本地  │      │  MySQL 数据库          │
│  - 机器人模拟对战     │      │                        │
│  改造后：             │      │  JWT 鉴权              │
│  - wx.request → API  │      │  WebSocket 实时对战     │
│  - 真实玩家对战       │      │                        │
└──────────────────────┘      └──────────────────────┘
```

## 二、技术选型

| 选型 | 理由 |
|------|------|
| **Node.js** | 与前端同语言，game-engine.js 可直接复用 |
| **Express** | 用户指定 |
| **Socket.IO** | 对局 3秒/轮 实时广播，REST 轮询延迟不可接受 |
| **MySQL** | 用户指定 |
| **mysql2** | Node.js MySQL 驱动（支持 Promise） |
| **jsonwebtoken** | 微信 openid → JWT token |
| **cors/helmet/morgan** | 标准 Express 中间件 |

## 三、数据库设计（共 9 张表）

### 3.1 users — 用户表

```sql
CREATE TABLE users (
  id            INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  openid        VARCHAR(64) NOT NULL UNIQUE,       -- 微信 openid
  nick_name     VARCHAR(64) NOT NULL DEFAULT '微信用户',
  avatar_url    VARCHAR(512) NOT NULL DEFAULT '',
  level         INT UNSIGNED NOT NULL DEFAULT 1,
  total_exp     INT UNSIGNED NOT NULL DEFAULT 0,
  total_income  INT UNSIGNED NOT NULL DEFAULT 0,
  coins         INT UNSIGNED NOT NULL DEFAULT 8820, -- 与前端 INITIAL_COINS 一致
  game_count    INT UNSIGNED NOT NULL DEFAULT 0,
  win_count     INT UNSIGNED NOT NULL DEFAULT 0,
  created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_openid (openid)
);
```

### 3.2 rooms — 房间表

```sql
CREATE TABLE rooms (
  id            INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  room_code     VARCHAR(10) NOT NULL UNIQUE,        -- 4位数字邀请码
  stage_id      INT UNSIGNED NOT NULL,              -- 景区 ID

  host_user_id  INT UNSIGNED NOT NULL,              -- 房主
  max_players   TINYINT UNSIGNED NOT NULL DEFAULT 10,
  created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  finished_at   DATETIME NULL,
  INDEX idx_status (status),
  INDEX idx_room_code (room_code)
);
```

### 3.3 room_players — 房间玩家表

```sql
CREATE TABLE room_players (
  id            INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,

  UNIQUE KEY uk_room_user (room_id, user_id),
  UNIQUE KEY uk_room_seat (room_id, seat)

```

### 3.4 game_sessions — 对局表

```sql

  id            INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,

  INDEX idx_status (status)
```

### 3.5 game_players — 对局玩家表

```sql

  id            INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
```

### 3.6 game_rounds — 回合记录表

### 3.7 game_results — 对局结果表

```sql

```

### 3.8 user_items — 用户物品表

```sql
CREATE TABLE user_items (
  id            INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  user_id       INT UNSIGNED NOT NULL,
```

### 3.9 coin_records — 金币流水表

```sql
CREATE TABLE coin_records (
  id            BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  user_id       INT UNSIGNED NOT NULL,
  amount        INT NOT NULL,                       -- 正数为收入，负数为支出
  balance_after INT UNSIGNED NOT NULL,              -- 变动后余额
  type          VARCHAR(32) NOT NULL,               -- 'game_reward','purchase','trade_buy','trade_sell'
  title         VARCHAR(128) NOT NULL DEFAULT '',
  created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_user_time (user_id, created_at DESC)
);
```

## 五、Socket.IO 实时事件

客户端连接后发送 `{ token: "jwt..." }` 鉴权。

### 客户端 → 服务端

| 事件 | 数据 | 说明 |
|------|------|------|
| `join_arena` | `{ sessionId }` | 进入对局房间（加入 Socket Room） |
| `leave_arena` | — | 离开对局 |
| `emote` | `{ emoteId }` | 发送表情 |

### 服务端 → 客户端

| 事件 | 数据 | 说明 |
|------|------|------|
| `game_tick` | `{ timeLeft, round, events[] }` | 每秒同步（含回合结算） |
| `score_update` | `{ playerId, score, delta }` | 分数变化 |
| `fortune_bag` | `{ opportunity }` | 福袋出现通知 |
| `fortune_result` | `{ playerId, result }` | 福袋开奖结果 |
| `emote_broadcast` | `{ fromUserId, emoteId }` | 表情广播 |
| `game_finished` | `{ sessionId }` | 对局结束通知 |

## 六、游戏流程（服务端驱动）

以前是前端 `game-store.js` 驱动整个游戏循环，现在改为：

```
1. 房主点击 "开始游戏"
   → POST /api/rooms/:id/start
   → 服务端创建 game_sessions
   → 调用 engine.createInitialPlayers() + engine.pairPlayers()
   → 广播 game_started 到房间所有玩家

2. 服务端 setInterval 每秒 tick：
   - timeLeft--
   - 每 3 秒调用 engine.playRound() 计算回合事件
   - 广播 game_tick（含分数变化、事件）
   - 到达 0 → 调用 engine.buildResult() 结算
   - 记录 game_results + 更新用户金币/经验

3. 客户端自动跳转结算页面
   → GET /api/games/:id/result
   → 显示排行
```

**关键决策：** 服务端全权驱动游戏循环，客户端只负责展示和用户交互（表情、福袋选择）。游戏规则引擎 `game-engine.js` 的纯函数可直接复用到服务端。

## 九、关键决策与说明

### 1. game-engine.js 复用

前端的 `utils/game-engine.js` 是纯函数（无副作用），可直接复制到服务端使用。这保证了**前后端规则一致**。需要复制的文件：
- `constants.js`（去掉 wx 相关常量）
- `game-engine.js`（核心）
- `investment.js`（福袋/投资逻辑）
- `progression.js`（等级计算）

### 2. 游戏循环在服务端

前端之前的 `game-store.js` 用 `setInterval` 每秒 tick，现在移到服务端。前端 `arena/index.js` 不再自己 tick，而是监听 Socket.IO 事件更新 UI。

### 3. 玩家离线处理

- 断线 → 服务端自动补机器人代替（复用 engine.createRobotPlayer）
- 重连 → 恢复监听游戏状态

### 4. 安全性考虑

- 所有 API 需 JWT 鉴权（除了 login）
- 商城购买服务端校验金币余额
- 房间操作校验权限（房主才能 start/解散）
- 防刷接口（同一用户不能同时加入多个房间）
