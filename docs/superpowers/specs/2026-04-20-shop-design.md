# 商城功能设计

**日期：** 2026-04-20

## 目标

在对局页新增商城入口，并提供一个独立商城页，让玩家可以使用“幸运金币”购买皮肤和宠物。购买结果、已拥有列表和当前装备状态都保存在本地；当前装备会回显到对局页自己的玩家节点上。

## 页面范围

### 对局页 `pages/arena/index`

- 保持现有页面根结构不变，继续遵守 `global-bg-image` → `app-header` → `page-content`。
- 在 `page-content` 内新增商城入口按钮，避免与系统状态栏、胶囊和头部安全区重叠。
- 按钮点击后跳转到独立商城页。
- 对局页只展示“我”的装备效果，不改变其他玩家节点逻辑。

### 商城页 `pages/shop/index`

- 新增独立页面，并加入路由。
- 页面根结构严格遵循项目硬规则：
  1. `global-bg-image`
  2. `app-header`
  3. `page-content`
- 页面内容包括：
  - 当前幸运金币余额
  - 分类切换：`皮肤` / `宠物`
  - 商品卡片列表
  - 商品状态按钮：`购买` / `已拥有` / `设为使用` / `使用中`

## 数据模型

新增本地商城仓库，使用 `wx.setStorageSync` / `wx.getStorageSync` 持久化，数据结构如下：

```js
{
  coins: 8820,
  ownedSkins: ['skin-default'],
  ownedPets: [],
  equippedSkinId: 'skin-default',
  equippedPetId: ''
}
```

说明：

- `coins` 复用当前产品语义中的“幸运金币”。
- 默认给一个可玩初始值 `8820`，与首页当前展示一致，避免首页和商城感知割裂。
- 默认拥有一个基础皮肤 `skin-default`，便于“装备”逻辑始终有兜底值。
- 宠物默认无装备。

## 商品方案

### 皮肤

皮肤不替换真人头像素材，优先通过对局页“我”的节点样式差异表现：

- 改变头像外框渐变
- 改变“我”名称标签底色
- 必要时增加轻量阴影或描边

这样可以避免额外改造用户头像体系，也能保证实现成本可控。

### 宠物

宠物在对局页“我”的节点旁增加一个小型装饰视图：

- 显示宠物图标/emoji
- 显示宠物名短标签
- 只挂在自己的节点上，不参与其他战斗逻辑

## 购买与装备规则

### 购买

- 只有余额足够时允许购买。
- 已拥有商品不可重复购买，也不会再次扣费。
- 金币不足时给出明确提示。

### 装备

- 只有已拥有商品才允许装备。
- 同类商品始终只允许一个处于使用中：
  - 皮肤只允许一个 `equippedSkinId`
  - 宠物只允许一个 `equippedPetId`
- 购买成功后不自动装备，避免误触导致当前装配被覆盖。

## 对局页展示规则

- 商城入口按钮固定在 `page-content` 内的安全区域，避免挡住底部导航。
- 自己的节点读取商城仓库：
  - 根据 `equippedSkinId` 映射不同 class
  - 根据 `equippedPetId` 决定是否展示宠物挂件
- 不修改现有投资机会弹层、表情面板、队伍连线或结算逻辑。

## 测试范围

优先覆盖商城仓库层，至少包含：

1. 首次初始化时返回默认状态
2. 购买商品会扣除金币并写入已拥有列表
3. 金币不足时购买失败且不改写状态
4. 重复购买已拥有商品不会重复扣费
5. 装备已拥有商品会更新当前装备

## 非目标

- 不接真实支付
- 不接服务器存档
- 不把商城入口同步到首页或房间页
- 不修改对局收益结算来源
